# Spec — Product specification

Project name: Lecture File Manager

Overview
--------
Allow users (instructors, students) to upload, organize, and share lecture PDF files. Provide lightweight file metadata, summaries, flashcards, and example tasks generated by an LLM.

Primary features
----------------
1. File management
   - Upload PDF files and store them on the server.
   - Organize files into Groups and SubGroups.
   - Reorder files and groups; provide sharing links.

2. Timeline tasks
   - Create timeline tasks with start/end dates and optional group association.

3. AI-assisted content
   - Generate summaries, flashcards, and example tasks from uploaded PDFs.
   - Generate a group-level topics checklist from aggregated file metadata.
   - All AI outputs should be produced in German and be as detailed as possible.

User stories (examples)
-----------------------
- As an instructor, I can upload a PDF and get a detailed, topic-focused German summary and a set of flashcards to distribute to students.
- As a student, I can browse files by group and download PDFs.
- As an admin, I can create groups and subgroups and reorder files to control presentation order.

Acceptance criteria (for key flows)
-----------------------------------
- Uploading a valid PDF creates a database entry and stores the file under /uploads with a secure stored_name.
- Summarize endpoint returns a German markdown list of topics, each with 3–6 deep bullet points.
- Flashcard endpoint returns one card per line in `Front :: Back` format (German), covering all topics.
- Generating tasks returns 3–5 unique, non-repetitive task prompts.
- All LLM calls go through llm.py and respect environment configuration (AZURE_OPENAI_API_KEY, AZURE_OPENAI_ENDPOINT).

Non-goals
---------
- No authentication or user accounts in the current scope (recommendation: add later).
- No production-grade file scanning pipeline implemented (recommendation: add virus scanning, content-type validation).
